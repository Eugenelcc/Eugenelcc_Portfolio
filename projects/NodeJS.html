Face Recognition & Detection

HealthApp Face Recognition & Detection user account

Background

Nanyang Polytechnic Year 3, course-specialization on Artificial Intelligence and machine learning. Working with a group of 3 people and tasked of using Healthcare as an Use cases on an Artificial Intelligence Project.

Applying 14 weeks lesson of AI Application Project module and work together to make a Flask webframe with individual API, Face Recognition& Detection, Calaroies calculator and Chatbot. I was tasked on setting up user accounts where user can register with their face to the system and authenticate them with face login. Using process flowchart to map up the process and how the frontend will communicate with the API to store face embedded and retrain the model.

Training the Model

Using existing pre-trained model Face Recognition Model for detecting any face available when on camera while also applying the Skit-learn K-nearest neighbor classifier to fetch the face embedding from the frontend.

During registration of user account, in the frontend webpage user will take images of their face up to 3 times for training and post to the API. The API will then detect for face, if no face is detected the face index will remain the same number until a face is detected. Once the 3 images is upload and send to the API for training, the model will search for any existing face storage and load it in the extracted facial embedding will be set as X while Y will be the numbers of users in the database. Using the KNN classifier to weight for number of neighbors and applying Ball-Tree algorithm for training and calculate the distance of the face embedding.

When user attempting to do face login, the webcam will capture the face and send to the API side, the API will detect the face embedding and reference with the training data to find the closest distance and return the result of the user name. The API will do a POST to the frontend and authenticate the user based on the result it receive.

So in conclusion ...

After going through multiple test and trials, the face recognition is not perfect there are some errors which results the model unable to detect the faces due to the lighting of the surroundings as well as low chance of success recognizing asian people due to similar facial characteristics traits, this could be the cause of having a pre-trained model when training they did not provide enough different type of asian facial training data. Improvement would be increasing the number of shots and image of user when doing registrations.
