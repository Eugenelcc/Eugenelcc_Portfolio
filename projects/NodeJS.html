Full Stack Development - NodeJS

Background

A Team project with group of 4 people, learning NodeJS and applying the skills into create a website using handlebreaks as a responsive page instead of using the usual HTML pages.

Using MySQL Workbench as a database to store and fetch relevant information to use, the group is split into specific tasks and management, Account management, Inventory management, website details and payment transactions using paypal sandbox

Account management packages involve are using Passport-Oauth2.0 for local and multi authentication, JWT token for creating web tokens for verificaiton and nodemailer to send out email to intended users.

My Tasks

My role in this project is to manage and setup the account management, setting up MySQL schema database for user managements as well as the inventory storage. When the user create an account, the password will be encrypt with hash for privacy and secruity reason, this can be seen in MySQL workbench which admin access to the database won't know the user information.

After registation,the user will require to veriy their account in their email based on the input they set during registration. There is a scrap email setup for the backend to automate and send email to user for verifications this is backen by using JWT Web token for the User to verify, once they click and verify the request, this will communicate with the database and set the verify from False to True allowing the user to login their account

There is also both Google and Facebook Oauth2.0 using passport for user to create an account based on either Google email or Facebook email instead of going through the manual process, this is also can be seen in MySQL database instead of saving the password, the backend will save either GoogleID or FacebookID and left the password option to be blank.

The Admin are given an admin account to access the Admin Dashboard where they can interact with user accounts, if they judge the user have been breaking the policy they can choose to delete the account which the backend will notify the deleted user to inform their account deletion

So in conclusion ...

This experience I have with Node JS is interesting and great insightful experience as I was able to explore on Google and Facebook Oauth2.0 services using Passport as well as using Nodemailer for sending email user for verification as well as Password reset. But there are also some bugs and fixes which require to be review and check on, there also some ethical involve as Admin are forbid to mess around the edit of user accounts as this can invade privacy and informations.
